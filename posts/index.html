Orin AGX 物理连接到主机 1.1 连转接线 使用Orin自己的USB-TypeC转接线。（图方便想用小米充电线失败）。USB连电脑，TypeC连Orin的电源键那一侧的C口，而不是电源线同侧。电源线同侧的C口是C口电源线。 这是连左侧C口，Orin进入Recovery状态，管理员powershell 显示。Recovery状态：长按中间键，然后按右键，先松右键，再送中间键。此时DP连接的显示屏黑屏。注意跟网上host是虚拟机的教程不同的点是，Win的PowerShell上显示的是APX，这可能会造成误解。 这是连左侧C口，Orin正常开机状态，显示屏显示开机。 连右侧C口时list中少一行，没有识别任何东西。 1.2 安装usbipd 管理员终端，安装usbipd-win工具，使得能够在WSL中使用USB。 手动安装usbipd，安装链接：Releases · dorssel/usbipd-win 点击下载 .msi的安装包，双击.msi文件。然后PowerShell检查是否有安装路径。 win上添加环境系统变量 C:\Program Files\usbipd-win\ 。 1.3 将USB绑WSL上 使用usbipd-win 连接USB设备 ref 。 终端开WSL wsl -d Ubuntu-22.04。 管理员终端： 共享目标设备 usbipd bind --busid 2-9 ， 验证设备共享状态 usbipd list， 附加USB设备 usbipd attach --wsl --busid 2-9 。 WSL中查看USB设备 lsusb。为了避免后面FLASH时候断连，可以加-a参数，用 usbipd attach -i 0955:7023 -w Ubuntu-22.04 -a 保持自动连接。 左图时主机终端显示，右图是WSL显示 Recovery状态的设备，应该在WSL中显示 NVIDIA Corp. Set Fail2ban in 3 Minutes

sudo apt install fail2ban # `sudo apt update` sudo cp /etc/fail2ban/jail.conf /etc/fail2ban/jail.local sudo vi /etc/fail2ban/jail.local # only edit local config file [DEFAULT] bantime = 10d # Ban duration: 10 days findtime = 1d maxretry = 3 # if 3 times fail in 1 day then ban ignoreip = 127.0.0.1/8 ::1 [sshd] enabled = true port = ssh filter = sshd logpath = %(sshd_log)s # or `logpath = /var/log/auth.log` sudo systemctl restart fail2ban sudo systemctl enable fail2ban # Enable auto-start on boot sudo systemctl status fail2ban.service sudo fail2ban-client status sshd # Check SSH jail status sudo tail -f /var/log/fail2ban.log # Monitor real-time bans sudo iptables -L -n | grep REJECT sudo cat /var/log/auth.log m-minute, d-day, none-second. search in vi: press /word then Enter. n for next and N for previous. ignoreip 127.0.0.1/8 refers to the IPv4 loopback address range. The /8 notation indicates this covers all addresses from 127.0.0.0 to 127.255.255.255. These addresses are used for local communications within the same machine. ::1 is the IPv6 loopback address, which serves the same purpose as 127.0.0.1 but for IPv6 networks. 多个 IP 地址之间使用空格分隔 ref: Fail2Ban入门教程（非常详细） - C语言中文网
